Пишем свой фреймворк - версия 2 - есть публичная и админская части, есть редактор БД, 
есть сортировка товаров по убыв/возр, любые отклонеия от контроллера, действия, 
или превышение пагинации, использование текста где ожидается пагинация или наоборот вызывают ошибку
404 (Класс ошибок находится в роутере)

index.php/admin.php - точки входа

ФАЙЛЫ:
route - управляет подключением контроллеров/моделей и вызывает ошибки
config - константы контроллера/модели/вида
bd - поключение к БД

КОНТРОЛЛЕРА всего 4: 
default - общий для таблицы товаров и категорий - разделение мметодами (действие - товары, 
         действие - категории и тд.)
tables - страница выбранного товара, отражает все категории товара, при выборе категории -  
         возврат в default но показать только товары и только этой категории
admin - контроллер редактирования БД, внутри подключен контроллер default для вывода таблиц публичной части.
out - контроллер выхода - завершает сессию и покидает админ-часть


ВИДЫ:
основные main.tpl.php и admin.tpl.php
остальные реализованы как слои


ОШИБКИ: 
1) вызов публичного контроллера в админ приводит к перекрытию вида со стороны публичной части
2) времено убрал кнопку сортировки в публичной части выбранной категории, в текущем проекте 
    сортировка устанавливается в самом публичном контроллере и сохраняется в отфильтрованном
